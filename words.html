FROM GAME THEORY TO BITCOIN

This isn't just an abstract game. These exact dynamics play out every day in Bitcoin mining:
• Consensus-Following = Build on the longest valid chain (majority rule) • Divergent Path = Build on a minority fork (alternative consensus choice) • Payoffs = BTC rewards scale with network alignment; divergent paths earn less as the consensus chain grows • Strategies = Different mining approaches (responsive to feedback, always-consensus, adaptive, etc.)
Bitcoin's design principle: the network follows the longest valid chain, so consensus-aligned strategies earn more over time.

BITCOIN BLOCK ECONOMICS

These payoffs reflect real Bitcoin mining economics based on fork consensus:
• Both Follow Consensus (R=2): Both miners earn 6.25 BTC + fees (full reward)
• You Follow Consensus (S=-1): You earn full reward, other miner's fork rejected
• You Build Minority Fork (T=3): Short-term reward from minority followers, but long-term loss as consensus chain grows
• Both Build Different Forks (P=0): Network splits, both chains earn reduced rewards

The key: Bitcoin's consensus mechanism (longest chain wins) makes following the majority consensus the most profitable strategy over time.

LONG-TERM INCENTIVES & DIFFICULTY

Bitcoin's design isn't based on altruism. It makes diverging from consensus more expensive than aligning with it:
• Difficulty Adjustment: Every 2016 blocks (~2 weeks), Bitcoin automatically adjusts mining difficulty to maintain constant block time. This ensures divergent chains fall behind faster.
• Cost to Diverge: Diverging costs the same hash power as consensus-following, but earns less because minority followers are fewer. Consensus chains accumulate faster.
• Network Coordination: The longest valid chain becomes the canonical chain. Blocks on divergent paths get orphaned, reducing rewards.
• Compound Advantage: After 780,000 blocks, consensus-aligned miners have earned full rewards. Divergent miners have earned from fewer followers and face increasing rejection.

The math is undeniable: consensus-aligned strategies accumulate more wealth in the long run.

FORK CHOICE RULE: LONGEST VALID CHAIN WINS

This game implements Bitcoin's actual fork choice rule:
• Both Consensus-Following: Both miners' blocks get accepted and validated. Network grows steadily.
• One Diverges: Divergent miner's minority fork accumulates slower. Consensus chain grows faster, making the divergent blocks orphaned. Consensus-following miner earns more.
• Both Diverge Differently: Network splits. Both chains grow slowly, accumulating fewer followers. Both earn less due to reduced network coordination.

This creates the emergent dominant strategy: consensus-aligned strategies earn more because the network coordinates around the longest valid chain.

NETWORK EFFECTS & REPUTATION

In Bitcoin, reputation isn't just a game score – it has real consequences:
• Trusted Nodes: Nodes with good reputation get prioritized block relay, reducing orphan rates.
• Suspicious Nodes: Nodes that attack get banned from mempool access, reducing their profitability.
• Permanent Blacklists: Once banned, nodes must change IP addresses to rejoin the network.
• Economic Incentives: Honest miners earn compounding rewards. Attackers face increasing costs.

This is why Responsive Validator dominates in the long run – it's not just about one block, but about building a sustainable mining business.

When block propagation is delayed, nodes don't all see the same chain at the same time. This creates an opportunity:

The race attack: You mine a block secretly. While others validate it, you start mining the next block yourself. When your block propagates, you might have a 2-block lead! This hidden mining (selfish mining) can be profitable if you have enough hash power.

If a node knows others will see its blocks with delay, it can exploit that delay by mining in secret. Suddenly, Divergent Miner becomes profitable even against Responsive Validator!

This is called: selfish mining.

If people can't see what you're doing, game theory shifts. Honest behavior is rewarded less. Selfish behavior is rewarded more.

Now the whole network spirals into distrust:

Because some miners can profit from selfish mining when latency is high, their share of the network grows. As dishonest miners become common, even honest miners feel pressured to attack to stay competitive.

And once everyone is attacking each other? Nobody can trust anybody. The network fractures. Different mining pools create different fork chains. Security degrades.

The core problem: when delays hide actions, attacks become profitable and trust collapses.

Bitcoin's solution: Make the rules transparent and immutable.

Every node enforces the exact same rules. Forever. Globally.

No central authority can change the game mid-play.

No miner can profit from rule-breaking because all nodes reject it.

Honesty becomes the only profitable strategy.

And that's how Bitcoin stays secure for 15+ years.

Made attacks visible: Any deviation from consensus is immediately detectable by anyone running a node.

Made punishment automatic: Invalid blocks are rejected. Cheaters gain nothing.

Made the game infinite: Miners face the same rules forever, so short-term attack gains don't justify long-term reputation loss.

By changing the game structure itself, Bitcoin made honesty the dominant strategy. Not because everyone is good, but because the rules make cheating unprofitable.

What the game is, defines what the players do.
Our problem today isn't just that people are losing trust,

That may seem cynical or naive -- that we're "merely" products of our environment -- but as game theory reminds us, we are each others' environment. In the short run, the game defines the players. But in the long run, it's us players who define the game.
